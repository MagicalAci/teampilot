## 输入数据结构定义

### 1. 总数据结构

AI 接收的输入为：**走神专注度图片**、**科目**、**时间**。推荐标准 JSON 如下（图片为 URL 或 base64 二选一，见回退口径）。

```json
{
  "focus_image": "https://example.com/focus-chart.png",
  "subject": "数学",
  "time_range": "2025-03-13 14:00-15:30"
}
```

### 2. 一级对象字段说明

| 字段 | 类型 | 是否必填 | 含义 | 来源 | 回退 / 备注 |
|---|---|---|---|---|---|
| `focus_image` | `string` | 是 | 走神/专注度相关图片的 URL 或 base64 | 上游上传/埋点 | 见下节 |
| `subject` | `string` | 是 | 科目名称 | 用户选择/业务传入 | 缺失时用占位「未指定科目」 |
| `time_range` | `string` | 否 | 学习时间段描述 | 用户选择/业务传入 | 缺失时用「未指定时间」 |

### 3. 数组项与嵌套对象

本能力为单层输入，无数组或嵌套对象。

### 4. 字段回退口径

- `focus_image`
  - 优先取：请求体中的 `focus_image`（URL 或 base64）
  - 缺失时：不调用 AI，直接返回错误码与文案「缺少专注度图片」
- `subject`
  - 优先取：`subject`
  - 缺失时回退：`"未指定科目"`
- `time_range`
  - 优先取：`time_range`
  - 缺失时回退：`"未指定时间"`

### 5. 输入红线

- `focus_image` 只表示「走神/专注度」相关图表或截图，不得用于其他类型图片的语义承诺。
- 图片内容仅用于生成总结文本，不得在总结中编造与图片无关的数值或结论。
- 科目、时间仅作上下文，不可被 AI 编造为未在输入中出现的科目或时间。

## AI策略

### 1. 策略概览

- 单阶段生成：根据「走神专注度图片 + 科目 + 时间」一次生成总结文本。
- 生成顺序：输入标准化 → 调用多模态/视觉模型 → 得到总结文本。
- 输出边界：仅输出一段总结文本，不输出 JSON 多字段（除非下游明确要求结构化）。
- 不允许：对图片中未出现的数据进行编造；输出与学习报告无关的内容。

### 2. 模型与平台配置

| 项 | 值 / 说明 |
|---|---|
| `chat_url` | 按实际多模态/视觉 API 填写 |
| `model` | 支持图像输入的模型（如 Doubao-Seed 多模态或等价） |
| `APPID` | 按平台填写 |
| `agent ID` | 按平台填写 |
| `thinking` | 可选 enabled |
| `response_format` | 文本（当前输出为总结文本） |
| `鉴权方式` | Bearer Secret Key / 按平台 |
| `环境说明` | fat / uat / pre / prod 按实际 |

### 3. 模块级策略

| 模块 | 目标 | 输入 | 输出 | 选择/逻辑 | 关键约束 |
|---|---|---|---|---|---|
| 总结生成 | 根据专注度图片+科目+时间生成一段总结 | `focus_image`, `subject`, `time_range` | `summary_text` | 单次调用视觉+文本模型 | 不编造数据、不越界科目/时间 |

### 4. 排序公式与规则

本能力无排序/筛选逻辑。

### 5. 字段回退口径（策略层）

与「输入数据结构定义」中回退口径一致：缺图则直接报错不调用；缺科目/时间用占位文案。

### 6. 结构化输出与降级策略

- 当前输出为**纯文本**总结，不强制 `json_schema`。
- 若后续需改为 JSON（如 `{"summary":"..."}`），再增加 response_format 与解析；当前以文本为主。

### 7. 业务红线

- 不得编造图片中未出现的数值、曲线或结论。
- 总结必须围绕「该时间段、该科目的学习/专注情况」，不得输出与学习报告无关的内容。
- 科目与时间仅来自输入，不得在总结中杜撰其他科目或时间。

## 提示词模块

### 1. 模块总表

| 模块 | 输入参数 | 输出参数 | 输出示例 | 前端映射 | prompt详情 |
|---|---|---|---|---|---|
| 总结生成 | `focus_image`, `subject`, `time_range` | `summary_text` | 见下节 | 学习报告总结文案 / API 字段 `summary` | 见本章第 3 节 |

### 2. 每个模块怎么写

- **输入参数**：`focus_image`（图片 URL 或 base64）、`subject`（ string ）、`time_range`（ string，可选）。
- **输出参数**：`summary_text`（ string ）。
- **输出示例**：`"本段时间数学学习专注度整体良好，中段有短暂走神，建议后续保持当前节奏并适当休息。"`
- **前端映射**：展示在学习报告「总结」区域；API 可统一用字段 `summary`。
- **prompt 详情**：见下。

### 3. Prompt 正文标准结构

```md
## 人设
你是学习报告助手，负责根据用户提供的「走神/专注度」图片以及科目、时间信息，生成一段简洁、客观的学习总结，语气中性、不夸大不编造。

## 任务
根据一张走神专注度相关图片，以及科目、时间范围，生成一段总结文本。该总结用于学习报告中的「总结」展示。

## 规则
1. 仅根据图片中可见的专注度、走神等信息进行描述，不编造图中未出现的数据或结论。
2. 总结中必须体现「科目」和「时间」上下文（若提供）。
3. 长度控制在 2–5 句话，适合直接展示在报告页。
4. 若图片无法识别或与专注度无关，说明「无法根据当前图片生成有效总结」。

## 输出规范
- 输出为一段纯文本，无需 JSON 包裹。
- 语言：中文。
- 不包含标题、列表符号，仅一段连贯文字。

## 严格禁止
- 禁止编造图片中未出现的数值、曲线、百分比或结论。
- 禁止输出与学习报告、专注度无关的内容。
- 禁止杜撰未在输入中出现的科目或时间。

## 思考链
1. 先理解图片内容：是否与专注度/走神相关、有无关键曲线或指标。
2. 结合科目与时间，组织一句总评。
3. 若有可识别的趋势或建议，用 1–2 句简要说明；否则仅做客观概括。
4. 输出前检查：无编造、无越界、长度适中。
```

### 4. Prompt 写作硬要求

已满足：人设仅角色与语气；任务仅本模块产出；规则可判定；输出规范为纯文本；严格禁止单独成段；思考链仅内部步骤。

### 2. 脚本入口与模块职责

| 类型 | 路径 | 作用 |
|---|---|---|
| 核心生成器 | `deliverables/learning-report-summary-script/summary_generator.py` | 接收输入、调用 Prompt、返回总结文本 |

| 测试文件 | `deliverables/learning-report-summary-script/test_summary.py` | 单元/集成验证 |

### 3. 运行顺序

1. 输入标准化：校验 `focus_image` 必填，补全 `subject` / `time_range` 回退。
2. 组装 Prompt：注入科目、时间与图片。
3. 调用模型客户端，传入图片与文本。
4. 解析模型返回，得到 `summary_text`。
5. 输出组装：返回 `{"summary": "..."}` 或纯文本（按接口约定）。

### 4. 输出示例

```json
{

}
```
