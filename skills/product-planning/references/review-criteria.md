# 审核标准

这份清单用于审核“别人使用 `/产品策划` 或 `/产品策划校验` 后提交的最终文档”，不是审核 Skill 包本身。

## 一票否决项

- [ ] 文档对象明确，没有把不同产品、端口、角色或阶段混成一件事
- [ ] 核心结论已经写清，不是资料堆砌或过程记录
- [ ] 关键判断有真实证据支撑，没有把猜测写成事实
- [ ] 文档已经可评审，没有明显未完稿状态

## 1. 定义与结论

- [ ] 一开头就能看清这份文档在回答什么产品决策问题
- [ ] 产品定位、目标用户、核心场景和要解决的问题定义清楚
- [ ] 范围取舍明确，知道这版要做什么、不做什么
- [ ] 结论型表达成立，读完能直接得到判断

## 2. 结构完整性

- [ ] 至少覆盖背景、目标、方案、范围、功能、流程或架构、风险或后续动作
- [ ] 总领层存在，能快速看懂整体产品结构、双端关系或模块关系
- [ ] 章节之间衔接顺，不互相打架

## 3. 证据与真实性

- [ ] 关键结论能对应到真实输入材料，如截图、体验记录、知识库、访谈或数据
- [ ] 猜测与事实分得开；无法确认的部分被明确标成待确认项

## 4. 图文与配图

- [ ] 该有的流程图、架构图、表格已经补齐
- [ ] 图的作用明确，且与附近正文口径一致

## 5. 可落地性与交接

- [ ] 核心功能拆解清楚，足以支撑后续设计、开发或评审继续往下走
- [ ] 版本、索引、图片路径和命名关系清楚，reviewer 能顺利接手

## 审核结论怎么判

- [ ] 通过：一票否决项全部通过，主要章节、证据、图文和版本同步无明显缺口
- [ ] 补充后通过：主判断正确，但缺少少量截图、配图、版本同步或局部论证
- [ ] 打回重做：产品定义错位、没有总领层、没有真实证据，或核心结论与材料对不上
